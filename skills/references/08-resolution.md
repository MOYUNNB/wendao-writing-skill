# 收线结算 · 深度方法论

---

## 模块目标

确保所有线索回收、角色弧光完成、主题呼应。

## 5 层追问链（详细版）

### 第 1 层：未结的线索

**问题模板**：
- 还有哪些未结的线索？
- 哪些是必须回收的（must）？
- 哪些是应该回收的（should）？
- 哪些是可回收可不回收的（optional）？

**优先级分类**：
- **must**：不回收读者会觉得烂尾
- **should**：回收了让故事更圆满
- **optional**：回收不回收都行

### 第 2 层：主角的代价

**问题模板**：
- 主角最终的代价是什么？
- 零代价的胜利是最无聊的

**关键测试**：
- 主角付出了什么不可逆的东西？
- 这个代价是否符合他的人物弧光？

**反模式**：
- ❌ 主角赢得一切，无任何损失
- ❌ 代价太大导致主角不像主角
- ✅ 主角赢了，但付出了符合人物逻辑的代价

### 第 3 层：反派的结局

**问题模板**：
- 反派的结局和他的动机一致吗？
- 还是为了"爽"强行写死？

**反派结局的 3 种选择**：
1. **物理死亡**：直接消失（最常见但风险大）
2. **意义死亡**：失去他追求的东西（更高明）
3. **转化**：从敌人变朋友/中立（需要铺垫）

**反模式**：
- ❌ 反派为死而死，无逻辑
- ❌ 反派突然降智
- ✅ 反派的结局反映他的逻辑和选择

### 第 4 层：主题呼应

**问题模板**：
- 主题有没有呼应开篇？
- 首尾之间形成了什么对照？

**首尾呼应的 3 个层次**：
- **场景层**：开篇场景 vs 结尾场景（如开篇下雨，结尾雨停）
- **人物层**：开篇主角 vs 结尾主角（成长对照）
- **主题层**：开篇主题 vs 结尾主题（深化或反转）

**反模式**：
- ❌ 主题在结局才出现，前文无铺垫
- ❌ 主题与开篇完全无关

### 第 5 层：重读奖励

**问题模板**：
- 如果读者翻回第一章重新看，会发现什么之前没注意到的？

**重读奖励**：
- 关键伏笔的呼应
- 人物台词的双关
- 场景的暗示
- 主题的呼应

**关键洞察**：好故事经得起重读。

## 关键检查清单

### 完结检查清单（5 项）

```
□ 角色弧光完成：每个主要角色从起点到终点的变化是否清晰？
□ 线索回收：所有 must-priority 线索都回收了？
□ 主题呼应：结局与开篇形成对照？
□ 主角代价：主角付出了符合人物逻辑的代价？
□ 反派逻辑：反派的结局符合他的动机？
```

### 结局前的必做事项

1. **列出所有未结伏笔**（从 SB 中提取）
2. **确认每个 must-priority 伏笔都有 proposed_resolution**
3. **设计主角的代价**（不可逆）
4. **设计反派的结局**（符合逻辑）
5. **设计主题呼应的场景**
6. **设计至少 1 个重读奖励**

## 完结的 5 种方式

### 1. 圆满结局（Happily Ever After）
- 主角达成目标 + 获得幸福
- 适用：纯爱、治愈系
- 注意：必须有代价

### 2. 苦涩结局（Bittersweet）
- 主角达成目标但付出了大代价
- 适用：成长类、史诗类
- 注意：代价必须感人

### 3. 开放式结局（Open Ending）
- 主角达成部分目标，留有想象
- 适用：留白美学
- 注意：不能太模糊

### 4. 反转结局（Plot Twist）
- 表面胜利，实际失败（或反之）
- 适用：悬疑、反转
- 注意：反转要符合逻辑

### 5. 循环结局（Circular）
- 结尾呼应开篇
- 适用：宿命类、史诗类
- 注意：呼应要巧妙

## SB 字段定义（resolution）

```typescript
resolution: {
  unresolved_threads: {
    id: string;
    priority: 'must' | 'should' | 'optional';
    proposed_resolution: string;
    target_position: string;   // 章节位置
  }[];
  main_ending: string;
  character_endings: {
    name: string;
    ending: string;
  }[];
  final_battle: {
    scale: string;             // 规模
    cost: string;              // 代价
    surprise: string;          // 意外
  };
  completion_checklist: Record<string, boolean>;
}
```

---

## 完成标准

- unresolved_threads.priority = 'must' 的全部有 proposed_resolution
- main_ending 已设计
- 每个主要角色有 ending
- completion_checklist 至少 4 项为 true

## 反模式警告

| 反模式 | 触发信号 | 警告语 |
|--------|---------|--------|
| 零代价胜利 | "主角轻松获胜" | 代价 = 故事的重量；没有代价的胜利读者不会记住 |
| 反派强行死 | "反派必须死" | 反派的结局要符合他的逻辑，不是为了爽 |
| 主题无铺垫 | 结局突然点题 | 主题要贯穿全文，不能突然出现 |
| 伏笔烂尾 | "我忘了这条伏笔" | 必须回收，至少用一句话提到 |
| 角色弧光中断 | 主角性格突然改变 | 主角的结局性格要与过程一致 |
| 重读无奖励 | 结尾没有"彩蛋" | 好的结局经得起重读 |

## 推荐问题

1. "我的书快写完了" → 进入 ⑧ 收线结算
2. "我想确认有没有漏掉的线索" → 生成未结伏笔清单
3. "主角的代价怎么设计？" → 进入第 2 层追问
4. "反派的结局怎么写？" → 进入第 3 层追问
5. "怎么呼应主题？" → 进入第 4 层追问
