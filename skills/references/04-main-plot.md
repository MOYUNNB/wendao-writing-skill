# 故事脊梁 · 深度方法论

---

## 模块目标

帮作者找到故事自己的节奏和骨架，而不是塞进某个现成框架。

## 5 层追问链（详细版）

### 第 1 层：核心矛盾是什么？

**问题模板**：
- 你的故事核心矛盾是什么？
- 用「谁，要干嘛，什么阻碍他」三要素说清

**关键测试**：一句话能让陌生人复述这个矛盾。

### 第 2 层：能撑住 100 万字吗？

**问题模板**：
- 这个矛盾能不能撑住 100 万字？
- 如果不能，怎么裂变出更深的层次？
- 主角的"主动出击"阶段何时开始？

**关键洞察**：
- 短篇：单层矛盾足够
- 中篇：2 层裂变
- 长篇（100 万+）：3 层以上裂变 + 多卷循环

### 第 3 层：有大翻转吗？

**问题模板**：
- 故事中间有没有一次大翻转？
- 它让主角从「被动应对」变成「主动出击」了吗？
- 翻转前后的主角，核心诉求改变了吗？

**关键测试**：翻转后，主角想要的"东西"本质改变了。

### 第 4 层：最低点在哪？

**问题模板**：
- 主角的最低点在哪？
- 他失去了什么？
- 这个失去是他自己的选择造成的吗？

**McKee 洞见**：至暗时刻必须是主角自己的错导致，否则读者不会产生情感共鸣。

### 第 5 层：30 万字拿什么留住读者？

**问题模板**：
- 读者追到第 30 万字的时候，你拿什么让他们继续追？
- 主角的目标在 30 万字时是什么？
- 与开篇的目标有什么不同？

## 6 种参考结构

### 1. 核心裂变法
- **核心逻辑**：一句话定核心矛盾 → 三维扩展 → 织冲突网
- **适用**：玄幻/都市/长篇爽文
- **来源**：网文大神核心裂变法

### 2. 三幕升级版
- **核心逻辑**：起步期 25% → 发展期 60% → 冲刺期 15%
- **适用**：通用长篇
- **细分**：按字数划分阶段

### 3. Fichtean 曲线
- **核心逻辑**：危机先行，不断升级的危机直到高潮
- **适用**：悬疑/惊悚/快节奏文
- **特点**：没有"平静的开局"，第一个字就有危机

### 4. 爽文循环
- **核心逻辑**：装 → 打脸 → 升级 → 换地图 → 重新循环
- **适用**：升级流/系统流/打脸文
- **关键**：每个循环必须有"换地图"信号

### 5. Arc 结构
- **核心逻辑**：一句话概要 → 10-30 话 Arc → 单话大纲
- **适用**：连载网文/韩式网文
- **特点**：每 Arc 有情绪高潮

### 6. Save the Cat 15-Beat
- **核心逻辑**：15 个精确节拍，每卷循环一轮
- **适用**：剧情流/有明确三幕结构的故事
- **节拍**：Opening Image → Final Image 等

## AI 如何推荐结构

**苏格拉底式（不直接指定）**：

```
问 1: "你的故事是先有一个冲突画面，还是先有一个角色？"
  → 先有冲突 → 推荐 Fichtean/核心裂变法
  → 先有角色 → 推荐 三幕升级版/Save the Cat

问 2: "读者追你的书，追的是什么？爽感、悬念、还是情感？"
  → 爽感 → 爽文循环/核心裂变法
  → 悬念 → Fichtean/Arc 结构
  → 情感 → Save the Cat/三幕升级版

问 3: "你写的是升级流还是剧情流？"
  → 升级流 → 爽文循环 + 3-5-10 节奏
  → 剧情流 → Arc 结构/Save the Cat
```

## 通用检查点（框架无关）

任何故事都必须回答 5 个问题：

```
① 起点: 故事从哪开始？主角初始状态？
② 翻转: 最关键的转折是什么？它怎么改变了一切？
③ 至暗: 主角的最低点？他失去了什么？
④ 终局: 故事怎么收束？主角用了什么"新方法"解决核心矛盾？
⑤ 节奏: 紧张和舒缓的交替模式是什么？
```

## 网文长篇适配

### 一卷 = 一个结构循环
- 一卷（30-50 万字）= 一个完整结构循环（不管用哪种框架）

### 长篇 = 多卷循环
- 长篇（100 万+ 字）= 多卷循环，每卷的翻转揭开更大格局

### 三级大纲
```
总纲（全书）
  └─ 卷纲（每卷核心冲突 + 结构循环）
       └─ 章纲（每章目标 + 冲突 + 钩子）
```

### 3-5-10 节奏法则
- 每 3-5 章一个小爽点
- 每 10 章一个大反转
- 网文通用节奏底线

## SB 字段定义（main_plot）

```typescript
main_plot: {
  controlling_idea: string;       // 必填，Value + Cause
  structure_type: string;          // 必填：core_fission | three_act | fichtean | shuangwen_loop | arc | save_the_cat | custom
  structure_detail: JSON;          // 根据 structure_type 存储对应数据
  checkpoints: {
    origin: string;                // 起点
    reversal: string;              // 翻转
    darkest: string;               // 至暗
    resolution: string;            // 终局
    rhythm_pattern: string;        // 节奏模式
  };
  outline: {
    master: string;                // 总纲
    volume: {
      volume_num: number;
      title: string;
      word_count: number;
      core_conflict: string;
      beat_summary: string;
    }[];
  };
}
```

---

## 完成标准

- structure_type 确定
- 5 个 checkpoints 全有
- outline 至少有 master + 第 1 卷的 volume

## 反模式警告

| 反模式 | 触发信号 | 警告语 |
|--------|---------|--------|
| 塞进框架 | "我要用 Save the Cat" | 框架是拐杖不是模具；选适合你故事的 |
| 无翻转 | 故事一帆风顺 | 读者需要"被吓一跳"的瞬间 |
| 至暗无因果 | 反派太强导致至暗 | 至暗必须是主角的选择造成的 |
| 节奏拖沓 | 50 章还没到中点 | 你的故事可能需要缩短或加速 |
| 大杂烩 | 多种结构混用 | 一种结构用透 > 三种结构都浮于表面 |

## 推荐问题

1. "我想梳理主线" → 进入 ④ 故事脊梁
2. "故事结构怎么选？" → 进入 6 种结构推荐
3. "我卡在大纲上" → 直接进入第 2 层追问（能否撑住 100 万字）
4. "我想知道节奏" → 进入 3-5-10 节奏法则
