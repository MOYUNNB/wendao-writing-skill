# 章节推演 · 深度方法论

---

## 模块目标

为每章定义目标、冲突、钩子，保证节奏不断裂。

## 5 层追问链（详细版）

### 第 1 层：这一章目标

**问题模板**：
- 这一章的目标是什么？
- 推进什么剧情？
- 揭示什么设定？
- 发展哪条人物线？

**关键测试**：作者能用一句话说清"这章要完成什么"。

### 第 2 层：阻碍

**问题模板**：
- 主角在这一章遇到了什么阻碍？
- 这个阻碍怎么和他的核心矛盾相关？

**关键测试**：阻碍必须与核心矛盾相关，否则是无效情节。

### 第 3 层：爽点

**问题模板**：
- 这一章的爽点在哪？
- 如果没有，怎么加？

**爽点类型**：
- 实力装逼：主角碾压对手
- 打脸反派：让看不起主角的人震惊
- 收获宝物：获得重要资源
- 情感共鸣：读者与角色共情
- 真相揭示：意外但合理的揭秘
- 反转打脸：表面弱实际强的展示

### 第 4 层：章末钩子

**问题模板**：
- 章末的钩子是什么？
- 读者为什么翻下一章？

**常见钩子**：
- 危险迫近："门外的脚步声越来越近"
- 信息悬念："那封信上的字迹让我心跳漏了一拍"
- 场景中断："他刚要开口，天花板塌了"
- 悬念揭示："我突然意识到，那个神秘人就是我父亲"
- 选择困境："我必须在三秒内做出选择"

### 第 5 层：5 章节奏检查

**问题模板**：
- 连续 5 章看下来，情绪有没有起伏？
- 是不是全是紧张或全是平淡？
- 紧张-舒缓-紧张-舒缓-紧张 的节奏是否合理？

**节奏类型**：
| 类型 | 比例 | 适用 |
|------|------|------|
| 紧张密集 | 100% | 战斗类、悬疑 |
| 紧张-舒缓交替 | 50-50 | 通用 |
| 慢热铺垫 | 前 30% 舒缓 | 剧情流 |
| 爆发型 | 前 80% 舒缓 + 后 20% 爆发 | 高潮章节 |

---

## 章节四段结构

```
[目标] → [推进] → [冲突] → [钩子]
```

每章必有的 4 个元素：

1. **目标（Goal）**：主角这一章要完成什么
2. **推进（Progress）**：向目标迈进了什么
3. **冲突（Conflict）**：遇到了什么阻碍
4. **钩子（Hook）**：章末的悬念

**反模式**：缺一个就少一个味道。

---

## 爽点间隔

**3-5-10 节奏法则**：
- 每 3-5 章一个小爽点
- 每 10 章一个大反转
- 网文通用节奏底线

**爽点密度计算**：
- 100 章 = 20-30 个小爽点 + 10 个大反转
- 50 章 = 10-15 个小爽点 + 5 个大反转

**反模式警告**：
- 连续 5 章无爽点 → 读者流失
- 连续 3 章全是高潮 → 读者疲劳

## 关键约束

### 章节字数（按类型）

| 类型 | 单章字数 |
|------|---------|
| 网文连载 | 2000-3000 字 |
| 中篇 | 5000-8000 字 |
| 长篇单行本 | 8000-15000 字 |

### 章节命名

- 网文：第 X 章 + 序号（无标题或简略标题）
- 出版：第 X 章 + 标题（标题要勾起好奇）

## SB 字段定义（chapter_plan）

```typescript
chapter_plan: {
  chapter: number;
  title: string;
  goal: string;          // 目标
  conflict: string;      // 冲突
  cool_point: string;    // 爽点
  end_hook: string;      // 章末钩子
  emotion: string;       // 情绪（紧张/舒缓/爆发）
  word_count: number;
  status: 'planned' | 'drafted' | 'revised';
}[]
```

---

## 完成标准

- 每章 4 项字段（goal / conflict / cool_point / end_hook）齐全
- 5 章内有爽点起伏
- 章末钩子明确

---

## 反模式警告

| 反模式 | 触发信号 | 警告语 |
|--------|---------|--------|
| 无目标章 | 一章没明确推进 | 这章没有目标，删了读者不会发现 |
| 无冲突章 | 一帆风顺 | 主角一天过完 5 章 → 读者睡觉 |
| 无爽点章 | 一直铺垫 | 网文读者要爽点，3 章无爽点就走了 |
| 章末无钩 | 章末是日常 | 章末不钩，下一章读者流失 30% |
| 节奏单一 | 全是紧张或全是舒缓 | 紧张-舒缓交替才有呼吸感 |

---

## 推荐问题

1. "我想规划第 X 章" → 进入 ⑥ 章节推演
2. "我的节奏有问题" → 进入第 5 层追问（5 章节奏检查）
3. "怎么写爽点？" → 直接给爽点类型选项
4. "章末钩子怎么写？" → 给常见钩子选项

## 短篇视角：短篇无此模块

> **触发条件**：length_type === "short"，用户提到"章节/章纲"时。

**短篇没有"章"的概念**——除非要发平台硬拆。基本单位是**节拍（Beat）**。

**替代方案**：
- 节拍设计 → 见 `references/short-beat-design.md`
- 500-1000 字节拍 4 要素：微目标 / 微冲突 / 微情绪 / 微钩子
- 情绪弧线：起点→终点，画图检查无"情绪平原"
- 中段沼泽避坑：节拍 3-4 必加微转折

**章节四段结构在短篇的对应**：
- 长篇：[目标] → [推进] → [冲突] → [钩子]（每章）
- 短篇：[微目标] → [微冲突] → [微情绪] → [微钩子]（每节拍）

**3-5-10 节奏法则不适用**：
- 长篇：每 3-5 章小爽点，每 10 章大反转
- 短篇：每个节拍必须有微钩子（500-1000 字一钩）
- 短篇不"攒"爽点，节拍即爽点

**苏格拉底引导（短篇版）**：
- 问 1：你的节拍数 = 总字数 / 600
- 问 2：每个节拍有 4 个 micro 字段吗？
- 问 3：中段（节拍 3-4）有微转折吗？
- 问 4：情绪弧线画出来有"平原"吗？
