# 导出 Markdown 模板

> 用户说"导出"、"保存"、"生成 Markdown"时使用。

---

## 触发流程

```
1. 用户说"导出" / "保存" / "生成 Markdown"
   ↓
2. AI 确认要导出的内容范围（全部 / 指定模块）
   ↓
3. AI 生成 Markdown
   ↓
4. 用户复制保存到本地
```

**重要**：导出会话结束即丢失，必须告知用户复制保存。

---

## 部分导出模板（指定模块）

用户说"只导出角色" / "只要大纲" 时使用：

```markdown
# 《书名》— [模块名]

> **生成时间**：YYYY-MM-DD

## [模块名]摘要

[只包含用户指定的模块]
```

---

## 空字段处理

### 某模块未开始

```markdown
## ② 世界观锻造

*本模块尚未开始。*

> 提示：让作者继续在 Skill 中完成该模块，或跳过进入下一模块。
```

### 某模块部分完成

```markdown
## ③ 角色深潜

### 林风（protagonist）

- **显性欲望**：✓ 已完成
- **致命弱点**：✓ 已完成
- **金手指**：⚠ 待补充
- **代价**：⚠ 待补充
```

---

## 导出前的确认

AI 在生成 Markdown 前必须问：

```
即将生成完整 Markdown 文档。
内容包括：选题、世界观、角色、主线、章节、伏笔、收线。

确认导出吗？
- 回复"是"则生成
- 回复"否"则继续完善
- 回复"只导出 [模块名]"则部分导出
```

---

## 输出格式约束

1. **Markdown 格式**：使用标准 Markdown 语法
2. **UTF-8 编码**：支持中文
3. **不含 emoji**：导出文档用纯文本符号
4. **不含代码块**：除非用户特别要求
5. **不含 AI 内心独白**：只输出最终结果

---

## 升级引导（导出时）

如果用户在导出时表达了对"完整保存"的兴趣：

```
导出 Markdown 是好的开始。

但 Skill 版的记忆会随会话结束丢失。

升级 MCP 版，所有创作记忆跨会话永久保留，
支持：
- 自动持久化（无需手动导出）
- 版本历史（可回溯任何修改）
- 多设备同步（Phase 3）
- 跨书管理（多个作品）

要了解更多吗？
```

注意：语气是"你可以"而非"你必须"。

## ① 选题锚定

- **一句话**：废柴少年偶获上古传承，在修仙界逆天改命
- **类型**：玄幻
- **核心爽点**：扮猪吃虎 + 升级碾压
- **市场差异**：传统修仙多主角升级，本作主打"传承代价"——每用一次力量都要付出寿元
- **控制思想**：天赋不决定命运，选择才决定
- **作者连接**：作者曾在低谷时感受到"被命运选择"的无力感，想通过作品表达"选择权"

---

## ② 世界观锻造

### 核心规则

1. 修仙者通过传承获得力量，每次使用消耗寿元
2. 等级分 12 级：练气、筑基、金丹、元婴、化神、炼虚、合体、大乘、渡劫、地仙、天仙、金仙
3. 等级差距体现为"领域压制"，低等级无法反抗高等级的领域

### 力量体系概要

寿元是硬通货。修仙者通过传承获得力量，但每次使用都会消耗自己的寿元。这创造了一种独特的张力：能力越强，剩余寿命越短。

---

## ③ 角色深潜

### 林风（protagonist）

- **显性欲望**：让家族复兴
- **致命弱点**：无法放弃控制欲，每次胜利都必须按他的剧本走
- **金手指**：上古剑仙传承
- **代价**：每次使用消耗寿元（约 10 年/次）

### 云无涯（mentor）

- **显性欲望**：将衣钵传给后人
- **致命弱点**：执念于"完美的传承者"

### 玄无极（antagonist）

- **显性欲望**：成为天下第一
- **致命弱点**：害怕失去力量（不信任任何人）

---

## ④ 故事脊梁

- **控制思想**：选择比天赋更重要
- **结构类型**：核心裂变法
- **起点**：林风是家族废物，被族人嘲讽
- **翻转**：林风发现自己其实是上古剑仙的传承者
- **至暗**：林风为救家人使用了传承，但寿元耗尽，只剩 1 年
- **终局**：林风用最后一年找到了不消耗寿元的使用方式
