# 导出 Markdown 模板

> 用户说"导出"、"保存"、"生成 Markdown"时使用。

---

## 触发流程

```
1. 用户说"导出" / "保存" / "生成 Markdown"
   ↓
2. AI 确认要导出的内容范围（全部 / 指定模块）
   ↓
3. AI 生成 Markdown
   ↓
4. 用户复制保存到本地
```

**重要**：导出会话结束即丢失，必须告知用户复制保存。

## 导出前的确认

AI 在生成 Markdown 前必须根据 length_type 问：

**长篇路径**：
```
即将生成完整 Markdown 文档。
内容包括：选题、世界观、角色、主线、章节、伏笔、收线。

确认导出吗？
- 回复"是"则生成
- 回复"否"则继续完善
- 回复"只导出 [模块名]"则部分导出
```

**短篇路径**：
```
即将生成完整 Markdown 文档。
内容包括：情绪锚点、意象与极简规则、角色（短篇版）、节拍设计、结尾类型学。

确认导出吗？
- 回复"是"则生成
- 回复"否"则继续完善
- 回复"只导出 [模块名]"则部分导出
```

### 短篇导出示例

```markdown
# 《被遗忘的信》创作档案 · 短篇

> **生成时间**：2026-06-25 14:30
> **基于**：问道写作 · 苏格拉底式引导（短篇路径）
> **作品类型**：短篇（5000 字，知乎体）

---

## ① 情绪锚点

- **主导情绪**：压抑中的希望
- **一句话钩子**：我是一名护士，但母亲临终前最后一封信始终没寄到
- **类型**：情感 / 现实
- **情绪演化**：从"压抑"到"压抑中的自我接纳"
- **反例**：这不是"悲伤"，是"压抑中还有希望"

### 导语（200-500 字）

我已经在医院走廊值了七年夜班。每晚 11 点，护士站会亮起一盏红灯——意思是"有未读的信"。没人知道这灯为什么亮，就像没人知道这信为什么永远不送出。直到那个雨夜，主治医生递给我一封没有署名的信，上面写着母亲的名字……

---

## ② 意象与极简规则

### 核心意象

1. 医院走廊的红灯
2. 旧信
3. 窗外的树

### 极简规则

这里的信件不送出，永远堆在护士站，等收信人主动来取。

---

## ③ 角色深潜（短篇版）

### 我（protagonist）

- **一句话人设**：我是一名护士，但母亲临终前最后一封信始终没寄到
- **显性欲望**：找到那封信
- **真实需求**：接受母亲已经离世
- **致命弱点**：无法主动索取（被动等信）
- **能否砍掉**：否（核心角色）

### 母亲（mentor · 未出场）

- **一句话人设**：临终前留了一封信，但从不主动联系
- **显性欲望**：让女儿来取信
- **致命弱点**：不愿意打扰女儿
- **能否砍掉**：否（信的源头）

---

## ④ 节拍设计

### 情绪弧线

平静 → 压抑 → 压抑+1 → 紧张 → 紧张+1 → 紧张+2 → 压抑中的释然 → 余韵

### 节拍列表

#### 节拍 1（约 700 字）
- **微目标**：建立"我"作为夜班护士的日常
- **微冲突**：护士站的红灯又亮了
- **微情绪**：平静
- **微钩子**：今天有一封新信
- **意象回声**：红灯、旧信
- **摘要**：夜班，护士站，红灯，7 年没动的信件

#### 节拍 2（约 600 字）
- **微目标**：揭示信的规则
- **微冲突**：信永远不送出
- **微情绪**：压抑
- **微钩子**：主治医生说"你妈当年也值夜班"
- **意象回声**：旧信
- **摘要**：规则揭示，主治医生暗示

...

### 结构类型

单一场景（一天一夜）

---

## ⑤ 结尾类型学

- **结尾类型**：留白
- **最后一击**：主角把那封信放在窗台（不寄出）
- **读者带走**：希望一直在，只是没注意
- **付费卡点**：N/A（文学类）
- **情绪呼应**：开篇的"压抑"在结尾变成"压抑中的自我接纳"
- **重读奖励**：开篇"没人知道这灯为什么亮"，重读时发现是"母亲一直在"

### 回收意象

- 红灯：开篇（普通）→ 结尾（母亲的在场）
- 旧信：开篇（压抑郁闷）→ 结尾（希望载体）
- 窗外的树：开篇（背景）→ 结尾（主角的视线停留）
```

## 输出格式约束

1. **Markdown 格式**：使用标准 Markdown 语法
2. **UTF-8 编码**：支持中文
3. **不含 emoji**：导出文档用纯文本符号
4. **不含代码块**：除非用户特别要求
5. **不含 AI 内心独白**：只输出最终结果
6. **短篇导出格式与长篇区分**：使用 emotion/short_world/short_characters/beats/short_ending 字段名

## 资料档案单独导出模板（v1.3.0+，仅长篇）

> 触发词：`导出研究`/`导出资料`/`导出档案`/`研究导出`/`资料导出`。
> 或主 SB 导出时含 ≥ 3 条带 source 的事实时自动建议。

### 触发流程

```
1. 用户说"导出研究"/主 SB 导出时 AI 建议
   ↓
2. AI 询问"是否同时导出研究档案.md"（如尚未询问）
   ↓
3. 用户答"是"或"只要档案"
   ↓
4. AI 扫描 SB 中所有带 source 的事实
   ↓
5. 按字段→方面映射归类（见下表）
   ↓
6. 生成研究档案.md（与主 SB 分开）
```
