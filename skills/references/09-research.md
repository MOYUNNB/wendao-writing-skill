## 摘要规范

每条事实输出格式：

```
- 事实：XXX
  来源：[来源名](URL) · 抓取于 YYYY-MM-DD
```

示例（宋朝服饰）：

```
- 事实：宋朝女官常服为褙子配裙
  来源：[中国古代服饰史 - 百度百科](https://baike.baidu.com/item/褙子) · 抓取于 2026-06-26
- 事实：宋朝男子常服为圆领袍，颜色按品级分
  来源：[宋代服饰文化 - 知乎专栏](https://zhuanlan.zhihu.com/p/xxx) · 抓取于 2026-06-26
```

## 来源可靠性分级

- **高**：维基百科、官方/学术站点、政府/官网、权威出版社、博物馆藏品页
- **中**：专业博客（知乎专栏、CSDN）、知名杂志、史料整理、机构公众号
- **低**：自媒体、论坛、来源不明、SEO 内容农场
- AI 应优先选**高/中**来源；**低来源事实必须打 `⚠ 来源弱` 标记**

## 写入 SB 字段规范

**世界观的资料**写到 `world.*` 字段；**人物的**写到 `characters[i].*`；**剧情的**写到 `main_plot.*`。

```json
{
  "world": {
    "costumes": [
      {
        "fact": "宋朝女官常服为褙子配裙",
        "source": {
          "name": "中国古代服饰史",
          "url": "https://baike.baidu.com/item/褙子",
          "fetched_at": "2026-06-26"
        }
      }
    ]
  }
}
```

**兼容性说明**：
- 旧版本（v1.2.x 及之前）`world.*` 字段值是 string
- v1.3.0+ 推荐值是 `array<{fact, source}>`，但旧的 string 值**仍兼容**
- AI 在更新字段时应**追加新事实**而非覆盖已有内容

## 控制规范（苏格拉底哲学保持）

1. **绝不超过 3 个问题/回应**
2. **绝不写正文**（搜来事实不变成段落描写）
3. **绝不编造来源**（搜不到就明说"未找到可靠来源"）
4. **总轮次 ≤ 3**（每个资料检索会话不超过 3 轮深挖）
5. **每轮最多 2 个方面**（控 token）
6. **每方面 3-5 条事实**（超过部分用户确认再要）
7. **用户确认才入 SB**（不入 SB 的事实只展示，不持久化）

## 与苏格拉底循环的关系

资料检索是**带联网能力的 LISTEN-RESPOND 增强**：

```
LISTEN → 用户确认"想搜资料"（显式答是）
   ↓
ASSESS → 当前 SB 已有设定：时代/地点/文化背景
   ↓
DECIDE → 选方面（9 选 N）→ 联网搜 → 摘要
   ↓
RESPOND → 展示摘要 → 问"哪些写进 SB？写到哪个字段？"
   ↓
UPDATE → 用户确认后写入对应字段
```

**6 种分支动作在资料检索中的映射**：
- **UNBLOCK**：用户说"随便搜搜" → 给 9 选 N 选择题
- **CONFIRM**：用户挑了事实 → 复述后问"写进哪个字段"
- **PIVOT**：搜了服饰但用户突然想看建筑 → 换方面
- **DEEPEN**：用户说"再深挖 X" → 1-2 轮深挖
- **GRADUATE**：搜完所有想搜的方面 → "资料检索完成！进 Phase B 吗？"
- **CHALLENGE**：搜来事实与已有 SB 矛盾 → "你说主角是宋朝人，但 X 是明朝才有的，怎么统一？"
