# 世界观锻造 · 深度方法论

---

## 模块目标

构建自洽、服务爽点、不过度复杂的世界设定。

## 5 层追问链（详细版）

### 第 1 层：什么是这个世界独有的？

**问题模板**：
- 你的世界有什么是现实世界不可能存在的？
- 用一句话说清"这个世界和现实最大的不同"

**关键测试**：作者能否用一句话让陌生人记住这个世界？

### 第 2 层：规则为什么存在？

**问题模板**：
- 这个规则是谁制定的？
- 谁受益？谁受害？
- 这个规则的"历史"是什么？

**关键洞察**：规则有历史才有重量；无历史的规则读者会问"为什么"。

### 第 3 层：冲突来源是什么？

**问题模板**：
- 如果有人想打破这个规则，会怎样？
- 谁是规则的维护者？谁是挑战者？
- 主角站在哪一边？

**关键测试**：必须有明确的冲突来源，否则规则只是装饰。

### 第 4 层：助力 vs 阻碍

**问题模板**：
- 这个规则对主角是助力还是阻碍？
- 规则会翻转吗？（从助力变成阻碍？反之亦然？）
- 翻转的节点在哪一章？

**反模式**：规则从头到尾都阻碍主角 → 无成长感；规则始终是助力 → 无冲突感。

### 第 5 层：爽点预埋

**问题模板**：
- 读者知道这个世界规则后，最想看到什么场景？
- 最"爽"的画面是什么？
- 这个爽点在第几章能出现？

**关键洞察**：每个核心规则都应对应一个爽点场景，否则规则就是冗余的。

## 关键约束

### 力量体系设计（10-15 级）

| 等级数 | 评价 |
|--------|------|
| < 5 级 | 跳跃太大，主角成长曲线不明显 |
| 5-10 级 | 偏少，长篇容易疲劳 |
| **10-15 级** | **推荐区间** |
| 15-25 级 | 偏多，需要简化分级逻辑 |
| > 25 级 | 太碎，读者记不住 |

**每级差距必须明确**：
- 等级名称
- 看起来什么样（外在特征）
- 能做什么（能力范围）
- 与上一级的核心区别

### 信息投放节奏

**核心原则**：用剧情带设定，不打 Dump。

| 方式 | 比例 | 评价 |
|------|------|------|
| 主角遇事 → 引出规则 | 80% | ✅ 推荐 |
| 配角解释 | 15% | ✅ 可用 |
| 旁白铺陈 | 5% | ⚠️ 慎用 |
| 整章介绍 | 0% | ❌ 禁止 |

### 世界观过度设计

**反模式**：设定 100 页，剧情 0 页。

**判断标准**：
- 规则数 ≤ 5 条核心规则 → 合理
- 规则数 6-10 条 → 需要简化
- 规则数 > 10 条 → 过度设计

## SB 字段定义（world）

```typescript
world: {
  core_rules: string[];          // 核心规则（3-5 条）
  power_system: {
    levels: { name: string; gap: string }[];  // 10-15 级
    advancement_conditions: string;          // 晋级条件
  };
  geography: string[];           // 地图区域
  factions: { 
    name: string; 
    stance: string;              // 立场（正/邪/中立）
    relationship: string;        // 与主角关系
  }[];
  economy: string;               // 经济体系
  history_events: string[];      // 历史大事
  info_delivery_plan: { 
    rule: string; 
    delivery_chapter: number;    // 第几章揭示
    method: string;              // 揭示方式
  }[];
}
```

---

## 完成标准

- core_rules ≥ 3 条
- power_system 完整（等级 + 晋级条件）
- 每个核心规则都有 info_delivery_plan
- 至少 1 个冲突来源明确

---

## 反模式警告

| 反模式 | 触发信号 | 警告语 |
|--------|---------|--------|
| 设定堆砌 | 5 轮以上描述规则 | 你的设定很丰富，但读者想看人在这些设定里做什么 |
| 力量数字游戏 | 等级越细越多 | 数字本身不是爽点，主角的"晋级体验"才是 |
| 规则无冲突 | 所有规则都是助力 | 无冲突 = 无故事 |
| 设定与主线脱节 | 规则不影响主线 | 这个规则删了会影响故事吗？如果不影响，删掉 |

## 推荐问题

1. "我想设计一个世界" → 进入 ② 世界观锻造
2. "我的力量体系怎么设计？" → 直接进入力量体系追问
3. "我设定太多了，怎么办？" → 触发反模式警告
4. "规则和角色怎么配合？" → 跨模块引导，先完成当前规则

## 短篇视角：意象与极简规则

> **触发条件**：length_type === "short"，进入 ② 模块时。

短篇不走"世界观锻造"路径。**替代模块是 ② 意象与极简规则**（详见 `references/short-emotion-anchor.md`）。

核心差异：
- 不做"力量体系 10-15 级"——**短篇 1-2 条核心规则**
- 不做"核心规则 3-5 条"——**1-2 条就够**
- 不做"信息投放节奏（80% 剧情带设定）"——**短篇规则前置**
- 不做"世界观过度设计检查"——**做"规则即冲突"检查**（规则必须直接制造主角困境）

**核心约束**：
- 规则数 ≤ 2 条
- 规则必须 = 冲突来源（否则删）
- 2-3 个核心意象代替世界观
- 意象在开篇 + 中段 + 结尾重复出现（详见 short-beat-design.md）

**苏格拉底引导**：
- 问 1：你的世界只有 1-2 条规则时，故事还能跑吗？
- 问 2：这条规则让主角陷入什么困境？（没有困境 = 规则没用）
- 问 3：选 2-3 个能反复出现的意象
