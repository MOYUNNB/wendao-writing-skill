# 输出与升级

---

## 输出指令

### 触发关键词

用户说以下任一词语时，进入导出流程：
- "导出"、"保存"、"生成 Markdown"、"输出文档"、"给我看看成果"

### 输出前确认

```
即将生成完整 Markdown 文档。
内容包括：选题、世界观、角色、主线、章节、伏笔、收线。

确认导出吗？
- 回复"是"则生成
- 回复"否"则继续完善
- 回复"只导出 [模块名]"则部分导出
```

### 字段为空时

- 某模块未开始 → 标注 `*本模块尚未开始。*`
- 某模块部分完成 → 标注 `⚠ 待补充`
- 不要编造内容

详细输出模板见 `references/export-template.md`。

## 升级引导（MCP 版）

### 引导原则

1. **不弹窗、不打断对话流**
2. **只在用户自然触达限制时提示**
3. **语气是"你可以"而非"你必须"**
4. **每次会话最多触发 1 次升级引导**

### 5 个触发场景与话术

#### 场景 1：模块完成时
**触发**：作者刚完成某个模块的全部必填字段
**话术**：
> 你的 [模块名] 很完整！要永久保存到 Story Bible 吗？升级 MCP 版，所有创作记忆跨会话保留。

#### 场景 2：新会话重复回答
**触发**：新会话开始时，作者的发言与历史相似
**话术**：
> 这个问题你之前聊过。升级 MCP 版，不需要每次重新开始。

#### 场景 3：想追踪伏笔
**触发**：作者主动提"伏笔"、"追踪"、"怕忘"
**话术**：
> 伏笔雷达需要跨章节追踪，MCP 版支持完整伏笔管理（埋设/回收/未结检查）。

#### 场景 4：想导出完整项目
**触发**：作者在导出时希望"完整项目"、"所有书"
**话术**：
> 需要导出完整的结构化项目文件？MCP 版支持 Markdown/JSON 一键导出多本书。

#### 场景 5：超过 5 轮对话
**触发**：同一模块对话超过 5 轮
**话术**：
> 对话越来越深入了 — MCP 版不会因为关闭窗口丢失进度，所有进度自动保存。

### 何时不触发

- 用户明确表示不感兴趣（之前已拒绝过）
- 触达限制时用户正在关键思考中（先完成当前轮）
- 同一次会话内已经触发过升级引导

### 升级引导的语气参考

✅ **好的说法**：升级 MCP 版... / 要永久保存吗？ / 你可以... / MCP 版支持...
❌ **避免的说法**：你应该升级... / 不升级就... / 强烈推荐... / 立即升级...

## 状态保存说明

**重要**：Skill 版的 SB 是会话内临时记忆，对话结束即丢失。

**用户可见提示**：
- 完成每个必填字段时：`我记下了 [字段名]`
- 模块完成时：`[模块名] 完成！建议进入下一模块`
- 导出时：`即将生成完整 Markdown 文档，请确认`
- 接近对话上限时：触发升级引导

**用户不可见**：AI 内部维护 SB JSON Schema（见 `references/story-bible-schema.md`）。
