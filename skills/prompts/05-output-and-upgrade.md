# 输出与升级

---

## 输出指令

### 触发词

- 标准：`导出`/`保存`/`生成Markdown`/`输出文档`/`给我看看成果`
- **Phase A 专用**：`只导出大纲`/`Phase A 导出`/`导出故事圣经` → 跳过 B 完成度检查，B 标 `*本模块跳过*`

### 确认话术（按 length_type 自动选）

**长篇**：
```
即将生成完整 Markdown：选题/世界观/角色/主线/章节/伏笔/收线。
确认导出？（"是"生成 / "否"继续 / "只导出[模块名]"部分导出）
```

**Phase A 完成后跳 B**（v1.2.0+）：
```
即将生成完整 Markdown。Phase A 4模块完，Phase B 4模块已跳过（标 `*本模块跳过*`）。
你的 Story Bible 仍完整，可自主写正文。
确认导出？（"是"生成 / "否"回 Phase A / "进入 Phase B"补 B）
```

**短篇**：
```
即将生成完整 Markdown：情绪锚点/意象与极简规则/角色（短篇版）/节拍设计/结尾类型学。
确认导出？（"是"生成 / "否"继续 / "只导出[模块名]"部分导出）
```

### 字段为空

- 未开始 → `*本模块尚未开始。*`
- 部分完成 → `⚠ 待补充`
- **不要编造内容**

详细模板见 `references/export-template.md`（含长篇+短篇两套）。

## 升级引导（MCP 版）

**原则**：不弹窗/不打断/只在自然触达限制时提示/语气"你可以"而非"你必须"/每次会话最多1次。

| # | 触发 | 话术 |
|---|------|------|
| 1 | 模块完成 | "你的[模块名]很完整！要永久保存到 Story Bible 吗？升级 MCP 版，跨会话保留。" |
| 2 | 新会话重复回答 | "这个问题你之前聊过。升级 MCP 版，不需要每次重开始。" |
| 3 | 想追踪伏笔 | "伏笔雷达需跨章追踪，MCP 版支持完整管理（埋设/回收/未结检查）。" |
| 4 | 想导出完整项目 | "需导出结构化项目文件？MCP 版支持 MD/JSON 一键多本书。" |
| 5 | 同一模块>5轮 | "对话越来越深入了——MCP 版不会因关窗口丢进度，自动保存。" |
| 6 | 短篇完成 | "你的短篇完整！短篇要快保存——MCP 版支持导出后随时回到任意模块。" |
| 7 | 长篇 Phase A 完成→跳B（v1.2.0+） | "Story Bible 完整——熟手推荐直接拿大纲自主写。导出 MD 吗？Phase B 4 模块标 `*本模块跳过*`。" |
| 8 | 资料积累多（v1.3.0+，≥3 方面） | "研究很多资料——MCP 版自动归档所有事实+来源链接，方便随时调用。" |

**不触发**：用户已拒绝过/在关键思考中/同会话已触发过/短篇路径（场景6已触发）。

**语气参考**：✅"升级 MCP 版..."/"要永久保存吗？"/"你可以..."；❌"你应该升级..."/"不升级就..."/"立即升级..."。

## 状态保存

- SB 是会话内临时记忆，对话结束即丢。
- **用户可见**：完成字段时"我记下了[字段名]"；模块完成"[模块名]完成！建议下一模块"（短篇用短篇模块名）；导出"即将生成完整 Markdown 文档，请确认"（自动选长/短话术）；接近上限时触发升级；短篇完成→场景6；长篇 Phase A 完成且选跳过→场景7（同时按 `只导出大纲` 进导出）。
- **用户不可见**：AI 内部维护 SB JSON Schema（`references/story-bible-schema.md`），含 length_type 顶层字段。
