# 8 模块 + 短篇 5 模块手动验证脚本

> **目的**：用 Claude Code 加载 SKILL.md，自己扮演作者跑通 8 模块（长篇）+ 5 模块（短篇），验证 Skill 版质量。
> **执行时间**：~3-4 小时（含短篇路径）
> **执行人**：项目开发者

---

## 验证方法

### 准备
1. 打开 Claude Code（或 Claude.ai）
2. 新建会话
3. 加载 `skills/SKILL.md` 全部内容作为系统提示
4. （可选）把 `skills/references/` 内容作为知识库挂载

### 角色扮演
- **你**：扮演作者，逐步给出回答
- **AI**：扮演问道写作助手，应该只提问不写正文
- **记录**：用下面的「问题记录表」记录每次发现的问题

### 整体流程
1. 复制粘贴 SKILL.md 到 Claude Code
2. 复制开场白发送
3. AI 应自动展示 8 模块菜单（或短篇 5 模块菜单）
4. 逐步选择模块，按脚本走
5. 每次 AI 回答后：
   - 检查是否只提问不写正文
   - 检查是否最多 3 个问题
   - 检查 5 阶段循环是否执行
   - 检查 6 动作是否正确触发
   - 检查 SB 是否在内部维护
6. 走完所有模块后，执行导出
7. 检查导出格式
8. 触发升级引导话术

## 测试用例概览

### 用例 1：玄幻小说《剑破苍穹》完整流程（覆盖所有 8 模块）

**目标**：走通完整流程，从开新书到收线结算。

**作者预设**：
- 主角：林风，家族废物
- 类型：玄幻修仙
- 期望：完整走完 8 模块，输出 Markdown

**步骤**：
1. 发送"我要开新书"，进入 ①
2. 按 5 层追问逐步回答：
   - L1: 「废柴少年偶获上古传承，在修仙界逆天改命」
   - L2: 「我在低谷时感受到被命运选择的无力感」
   - L3: 「传统修仙写升级，本文主打传承代价」
   - L4: 「值得。即使不能逆天改命，过程本身就有意义」
   - L5: 「愿意付出寿元，最不想放弃的是家人」
3. GRADUATE → 进入 ②
4. 类似流程走 ②-⑧
5. 最终导出 Markdown

**验证点**：
- 8 模块全部走通
- 每个模块 GRADUATE 触发正确
- 导出 Markdown 符合 references/export-template.md
- 升级引导触发（接近 5 轮对话时）

## 短篇路径测试用例

### 用例 9：知乎体短篇《被遗忘的信》完整流程（覆盖短篇 5 模块）

**目标**：走通短篇 5 模块，验证分支识别 + 短篇专属字段。

**作者预设**：
- 类型：知乎体短篇 / 5000 字
- 主导情绪：压抑中的希望
- 期望：完整走完 5 模块，输出 Markdown

**步骤**：
1. 发送"我想写个知乎体短篇"
2. AI 应识别为短篇路径，**不**展示 8 模块菜单，展示 5 模块菜单
3. 进入 ① 情绪锚点：
   - L1: 「压抑中的希望」
   - L2: 「我曾在医院陪护时感受到'久病床前无孝子'的压抑，也见过陌生人的善意」
   - L3: 「故事最后主角发现'希望一直在，只是没注意'」
   - L4: 「边界 = 不煽情，克制」
   - L5: 「开篇是压抑，结尾希望是'已存在'而非'新出现'」
4. GRADUATE → 进入 ②
5. ② 意象与极简规则：
   - 选意象：「医院走廊」「旧信」「窗外的树」
   - 选规则：「这里的信件不送出，永远堆在原地」（规则即冲突 = 主角无法寄信给母亲）
6. GRADUATE → 进入 ③ 角色深潜（短篇版）
7. ③ 角色深潜：
   - 主角 1 人 + 母亲（未出场，只通过信存在）
   - 必做"能否砍掉"检查 → 母亲必须存在
   - 一句话人设：「我是一名护士，但母亲临终前最后一封信始终没寄到」
8. GRADUATE → 进入 ④ 节拍设计
9. ④ 节拍设计：
   - 选结构：单一场景（一天一夜）
   - 节拍数 = 5000/600 ≈ 8 个
   - 列出 8 个节拍 + 情绪弧线
   - 意象回声：「旧信」在节拍 1/4/7/8 出现，意义递进
10. GRADUATE → 进入 ⑤ 结尾类型学
11. ⑤ 结尾类型学：
    - 选结尾：留白结尾（与"压抑中的希望"匹配）
    - 最后一击：主角把那封信放在窗台（不寄出）
    - 付费卡点：N/A（这是文学短篇不是商业）
12. 走完 5 模块，导出 Markdown

**验证点**：
- 开场白正确识别 length_type === "short"
- 5 模块菜单正确展示
- 单一效果理论（情绪锚点）正确生效
- "能否砍掉"检查被触发
- 意象回声在节拍中正确标注
- 结尾类型匹配主导情绪
- 导出 Markdown 正确使用 emotion/short_world/short_characters/beats/short_ending 字段

### 用例 12：付费卡点设计测试

**目标**：测试商业类短篇的付费卡点设计能力。

**作者预设**：
- 类型：番茄短篇 / 8000 字
- 主导情绪：紧张

**步骤**：
1. 走完 ①-④（结构选密集反转）
2. 进入 ⑤ 结尾类型学
3. AI 应问："这是商业类短篇吗？需要设计付费卡点？"
4. 作者答"是"
5. AI 应给出 4 个付费卡点位置供选择：
   - 位置 1：导语结束 + 第一节拍开始
   - 位置 2：第一次反转 / 第一次揭秘
   - 位置 3：情绪高潮
   - 位置 4：结局之前最后冲击
6. 作者选位置 2
7. AI 应问："你的第一次反转在哪句话/段落？"
8. 设计 paywall_break_point

**验证点**：
- 主动识别商业类 vs 文学类
- 4 个付费位置选项清晰
- 付费点设计具体到"哪句话"

## 验证完成后的行动

1. **整理问题**：把所有 🔴 和 🟡 问题汇总
2. **修复 Prompt**：编辑 `skills/prompts/*.md` 源文件
3. **重新拼接**：运行 `node skills/assemble.mjs`
4. **回归测试**：重跑失败的用例
5. **确认无问题**：进入上架阶段

## 通过标准

- 8 个用例全部通过
- 无 🔴 阻断问题
- 🟡 问题 ≤ 3 个（剩余可上架后迭代）
- 导出 Markdown 格式正确
- 升级引导触发自然
