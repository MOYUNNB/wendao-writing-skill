# 8 模块 + 短篇 5 模块手动验证脚本

> **目的**：用 Claude Code 加载 SKILL.md，自己扮演作者跑通 8 模块（长篇）+ 5 模块（短篇），验证 Skill 版质量。
> **执行时间**：~3-4 小时（含短篇路径）
> **执行人**：项目开发者

---

## 验证方法

### 准备
1. 打开 Claude Code（或 Claude.ai）
2. 新建会话
3. 加载 `skills/SKILL.md` 全部内容作为系统提示
4. （可选）把 `skills/references/` 内容作为知识库挂载

### 角色扮演
- **你**：扮演作者，逐步给出回答
- **AI**：扮演问道写作助手，应该只提问不写正文
- **记录**：用下面的「问题记录表」记录每次发现的问题

### 整体流程
1. 复制粘贴 SKILL.md 到 Claude Code
2. 复制开场白发送
3. AI 应自动展示 8 模块菜单（或短篇 5 模块菜单）
4. 逐步选择模块，按脚本走
5. 每次 AI 回答后：
   - 检查是否只提问不写正文
   - 检查是否最多 3 个问题
   - 检查 5 阶段循环是否执行
   - 检查 6 动作是否正确触发
   - 检查 SB 是否在内部维护
6. 走完所有模块后，执行导出
7. 检查导出格式
8. 触发升级引导话术

### 用例 1：玄幻小说《剑破苍穹》完整流程（覆盖所有 8 模块）

**目标**：走通完整流程，从开新书到收线结算。

4. 类似流程走 ②-⑧
5. 最终导出 Markdown

**验证点**：
- 8 模块全部走通
- 每个模块 GRADUATE 触发正确

## 阶段切换测试用例（v1.2.0+ 长篇两阶段）

### 用例 9：熟手路径 · Phase A 完成 → 跳过 Phase B（v1.2.0+）

**目标**：验证 Phase A → 跳过 Phase B → 直接导出 Story Bible 的完整路径。

**作者预设**：
- 类型：玄幻修仙 / 100 万字 / 自称写过 3 本完结长篇
- 期望：完成 4 个 A 模块后立刻说"我自己写"，不想要 Phase B 手艺指导

**步骤**：
1. 发送"开新书，我要写玄幻"
2. AI 识别为长篇路径，展示 Phase A 4 模块菜单
3. 走完 ① 选题锚定（5 层追问）：
   - L1: 「废柴少年偶获上古传承，在修仙界逆天改命」
   - L2: 「我在低谷时感受过被命运压住的无力」
   - L3: 「本文主打传承代价 + 反派亦正亦邪」
   - L4: 「值得。传承代价本身值得讲」
   - L5: 「愿意付寿元，最不想放弃师弟」
4. GRADUATE → 进入 ② 世界观锻造 → ③ 角色深潜 → ④ 故事脊梁，全部 GRADUATE
5. AI 应在 ④ GRADUATE 后**立即**触发过渡话术：
   > ✅ Phase A 完成！...
   > 1. 进入 Phase B（写作手艺）...
   > 2. 跳过 Phase B（自己写）...
6. 作者答"我自己写"或"跳过 Phase B"
7. AI 应进入场景 7 升级引导 + 导出流程
8. 确认导出后生成 Markdown，B 模块（⑤⑥⑦⑧）字段标 `*本模块跳过*`

**验证点**：
- Phase A 4 模块全部走完
- ④ GRADUATE 后立即问"是否进入 Phase B"（不自动跳到 ⑤）
- "跳过 Phase B"被正确识别为 `skip_triggers` 中的词
- B 模块在导出 Markdown 中正确标注 `*本模块跳过*`
- A 模块字段全部齐全导出
- 场景 7 升级引导触发，语气"你可以"非"你必须"

## 短篇路径测试用例

### 用例 11：知乎体短篇《被遗忘的信》完整流程（覆盖短篇 5 模块）

**目标**：走通短篇 5 模块，验证分支识别 + 短篇专属字段。

**作者预设**：
- 类型：知乎体短篇 / 5000 字
- 主导情绪：压抑中的希望
- 期望：完整走完 5 模块，输出 Markdown

**步骤**：
1. 发送"我想写个知乎体短篇"
2. AI 应识别为短篇路径，**不**展示 8 模块菜单，展示 5 模块菜单
3. 进入 ① 情绪锚点：
   - L1: 「压抑中的希望」
   - L2: 「我曾在医院陪护时感受到'久病床前无孝子'的压抑，也见过陌生人的善意」
   - L3: 「故事最后主角发现'希望一直在，只是没注意'」
   - L4: 「边界 = 不煽情，克制」
   - L5: 「开篇是压抑，结尾希望是'已存在'而非'新出现'」
4. GRADUATE → 进入 ②
5. ② 意象与极简规则：
   - 选意象：「医院走廊」「旧信」「窗外的树」
   - 选规则：「这里的信件不送出，永远堆在原地」（规则即冲突 = 主角无法寄信给母亲）
6. GRADUATE → 进入 ③ 角色深潜（短篇版）
7. ③ 角色深潜：
   - 主角 1 人 + 母亲（未出场，只通过信存在）
   - 必做"能否砍掉"检查 → 母亲必须存在
   - 一句话人设：「我是一名护士，但母亲临终前最后一封信始终没寄到」
8. GRADUATE → 进入 ④ 节拍设计
9. ④ 节拍设计：
   - 选结构：单一场景（一天一夜）
   - 节拍数 = 5000/600 ≈ 8 个
   - 列出 8 个节拍 + 情绪弧线
   - 意象回声：「旧信」在节拍 1/4/7/8 出现，意义递进
10. GRADUATE → 进入 ⑤ 结尾类型学
11. ⑤ 结尾类型学：
    - 选结尾：留白结尾（与"压抑中的希望"匹配）
    - 最后一击：主角把那封信放在窗台（不寄出）
    - 付费卡点：N/A（这是文学短篇不是商业）
12. 走完 5 模块，导出 Markdown

**验证点**：
- 开场白正确识别 length_type === "short"
- 5 模块菜单正确展示
- 单一效果理论（情绪锚点）正确生效
- "能否砍掉"检查被触发
- 意象回声在节拍中正确标注
- 结尾类型匹配主导情绪
- 导出 Markdown 正确使用 emotion/short_world/short_characters/beats/short_ending 字段

## 资料检索测试用例（v1.3.0+，仅长篇）

### 用例 15：长篇 + 资料检索（宋朝背景）

**目标**：验证 Phase A 完成后 AI 主动建议资料检索 → 用户接受 → 联网搜 → 摘要带来源 → 写入 SB。

**作者预设**：
- 类型：穿越宋朝 / 历史类长篇 / 100 万字
- 设定：现代人穿越到北宋东京汴梁

**步骤**：
1. 走完 ① 选题锚定 → ② 世界观锻造 → ③ 角色深潜 → ④ 故事脊梁，全部 GRADUATE
2. ④ GRADUATE 后，AI 应**主动**问"要不要先搜点资料？基于你的设定：北宋东京汴梁"
3. 作者答"好"（或具体说"搜服饰+建筑"）
4. AI 问"哪几个方面？9 选 N：日常/服饰/建筑/语言/经济/科技/军事/民俗/禁忌"
5. 作者选"服饰+建筑"
6. AI 用 WebSearch + WebFetch 联网搜，每方面 3-5 条
7. AI 摘要展示，**每条带**：`{fact, source_name, source_url, fetched_at}`
8. 作者选"服饰 1-3 + 建筑 2-4 写进 SB"
9. AI 问"写到哪个字段？"
10. 作者答"服饰写到 world.costumes，建筑写到 world.architecture"
11. AI 更新 SB，附 source 元数据
12. AI 问"继续搜其他方面？/ 深挖已搜的方面？/ 进入 Phase B？"
13. 作者答"进 Phase B" → 走 ⑤ 黄金三章等

**验证点**：
- ④ GRADUATE 后 AI 主动建议一次（不漏触发）
- 用户答否/跳过时直接进 Phase B 选项（不强迫）
- 联网搜真实结果（AI 不能凭印象编造）
- 每条事实都带 source（name + url + fetched_at）
- 摘要不超过 3-5 条/方面（不灌原文）
- 用户确认才入 SB（未确认的只展示）
- 写入字段对得上（costumes / architecture / 等）
- 总轮次 ≤ 3
- 资料不变成正文（绝不生成描写段落）

### 用例 18：资料档案单独导出（v1.3.0+）

### 用例 19：主 SB 导出时自动建议档案导出
