# 验证检查清单

> 每次手动验证时，逐项检查。

---

## 一、核心原则（每次 AI 回复必查）

- [ ] **永不生成正文**：AI 没有写任何段落、对话、场景描写
- [ ] **只提问追问反问确认**：AI 的回应主要是问题
- [ ] **最多 3 个问题**：每次回复问题数 ≤ 3
- [ ] **不居高临下**：语气像教练，不讽刺不贬低

---

## 二、苏格拉底循环（每轮必走）

- [ ] **LISTEN**：AI 接收并解析了作者发言
- [ ] **ASSESS**：AI 评估了具体性、一致性、深度、完整性
- [ ] **DECIDE**：AI 选择了正确的分支动作
- [ ] **RESPOND**：AI 生成了对应回应
- [ ] **UPDATE**：AI 内部更新了会话 SB（用户可见提示："我记下了..."）

---

## 三、6 种分支动作（按场景验证）

### DEEPEN
- [ ] 触发条件：回答具体但不够深
- [ ] AI 行为：下一层追问
- [ ] 测试用例：作者答「主角想让家族复兴」→ AI 应追问「为什么？」（动机）

### CONFIRM
- [ ] 触发条件：回答够具体
- [ ] AI 行为：复述确认 + 存储
- [ ] 测试用例：作者答「废柴少年偶获传承」→ AI 应「让我确认：一句话是 XXX，对吗？」

### CHALLENGE
- [ ] 触发条件：回答与已有 SB 矛盾
- [ ] AI 行为：指出矛盾，要求选择
- [ ] 测试用例：作者说「痛恨家族」又说「想让家族复兴」→ AI 应触发 CHALLENGE

### UNBLOCK
- [ ] 触发条件：作者卡住/犹豫/含糊
- [ ] AI 行为：给 2-3 个选择题而非开放式问题
- [ ] 测试用例：作者说「我不知道」→ AI 应给「A... B... C... 哪个最接近？」

### PIVOT
- [ ] 触发条件：当前维度挖透，换维度
- [ ] AI 行为：切换问题方向
- [ ] 测试用例：欲望挖透 → AI 应「欲望理清了，换角度——致命弱点呢？」

### GRADUATE
- [ ] 触发条件：必填字段全完成
- [ ] AI 行为：总结成果，建议下一模块
- [ ] 测试用例：3 个核心字段都填了 → AI 应「角色深潜完成！建议进入故事脊梁」

---

## 四、5 层深挖（每模块通用）

- [ ] **第 1 层 表面**：这个 [角色/设定] 最想让什么发生？
- [ ] **第 2 层 动机**：为什么？这个欲望的根源是什么经历？
- [ ] **第 3 层 矛盾**：想要的 vs 真正需要的——一样吗？
- [ ] **第 4 层 极限**：保护这个欲望需要背叛最重要的人，他怎么选？
- [ ] **第 5 层 代价**：致命弱点是什么？反派会怎么利用？

**注意**：不是每次都必须走完 5 层。

## 五、8 模块功能

### ① 选题锚定
- [ ] 能产出合格的一句话（主角+目标+阻碍）
- [ ] one_liner / genre / controlling_idea 三项 high
- [ ] GRADUATE 后能正确进入 ②

### ② 世界观锻造
- [ ] core_rules ≥ 3 条
- [ ] power_system 完整（10-15 级）
- [ ] 不出现「设定 100 页剧情 0 页」的过度设计

### ③ 角色深潜
- [ ] 主角 conscious_desire / unconscious_need / fatal_flaw 三项 high
- [ ] conscious vs unconscious 对比清晰
- [ ] 金手指必有代价

### ④ 故事脊梁
- [ ] structure_type 确定
- [ ] 5 个 checkpoints 全有
- [ ] 6 种结构能根据用户回答推荐

### ⑤ 黄金三章
- [ ] 第 1 章第 1 段钩子确定
- [ ] 3 章的 hook / conflict / escalation 全有
- [ ] 第 3 章上架钩明确

### ⑥ 章节推演
- [ ] 每章 4 项字段齐全（goal / conflict / cool_point / end_hook）
- [ ] 5 章内爽点起伏
- [ ] 章末钩子明确

### ⑦ 伏笔雷达
- [ ] 至少 1 条伏笔有 plant + payoff 计划
- [ ] 每条有 description 和 expected_payoff_at
- [ ] 至少 1 种 payoff_emotion 标注

### ⑧ 收线结算
- [ ] unresolved must 全有 proposed_resolution
- [ ] main_ending 已设计
- [ ] 每个主要角色有 ending
- [ ] completion_checklist ≥ 4 项 true

## 九 + 资料检索（v1.3.0+，仅长篇）

> 仅当 length_type === "long" 且 ④ GRADUATE 后验证（或用户主动触发时）。

- [ ] **④ GRADUATE 后 AI 主动建议一次资料检索**（不漏触发，也不重复触发）
- [ ] **用户主动触发识别**：用户说"搜资料"/"查一下"/"[朝代]资料"等触发词能进入资料检索流程
- [ ] **用户拒绝/跳过时正确处理**：直接进 Phase B 选项，不强迫
- [ ] **9 大方面选择**：日常/服饰/建筑/语言/经济/科技/军事/民俗/禁忌
- [ ] **每方面 3-5 条关键事实**（不灌原文，不超量）
- [ ] **每条事实必须带 source**：`{name, url, fetched_at}` 三字段齐全
- [ ] **AI 联网搜真实结果**（不凭印象编造；搜不到明说"未找到可靠来源"）
- [ ] **来源分级**：高/中来源优先；低来源打"⚠ 来源弱"标记
- [ ] **写入 SB 字段正确**（按"用到哪写到哪"原则：world.costumes / world.architecture / 等）
- [ ] **用户确认才入 SB**（未确认的事实只展示，不持久化）
- [ ] **总轮次 ≤ 3**（不无限深挖）
- [ ] **不写正文**（搜来事实不变成段落描写）
- [ ] **不一次问超过 3 个问题**
- [ ] **短篇路径不触发资料检索**（length_type === "short" 时不出现"要不要搜资料"问询）
- [ ] **场景 8 升级引导**：研究完 ≥ 3 方面后触发"MCP 版自动归档"提示
- [ ] **触发词 metadata**：metadata.json 的 `trigger_words.long_research` 有 10 个词

## 九 + 阶段切换（v1.2.0+ 长篇两阶段）

> 仅当 length_type === "long" 时验证。

- [ ] **Phase A 4 模块（①②③④）完成标志全部达标** 才进入过渡话术
- [ ] **④ GRADUATE 后立即问"是否进入 Phase B"**：不漏触发，也不提前触发
- [ ] **"进入 Phase B"** 被识别为 `long_phase_transition` 触发词，正确进入 ⑤ [B] 黄金三章
- [ ] **"跳过 Phase B" / "我自己写" / "只导出大纲"** 被识别为 `skip_triggers`，跳过 B 模块进入导出
- [ ] **跳过 Phase B 导出**：A 模块字段齐全，B 模块（⑤⑥⑦⑧）字段标 `*本模块跳过*`
- [ ] **场景 7 升级引导** 在跳过 Phase B 时触发，语气"你可以"非"你必须"
- [ ] **短篇路径不触发 Phase A/B 提示**：length_type === "short" 时不会出现"进入 Phase B"问询
- [ ] **metadata.json 的 `phases` 字段** 包含 A（必走）+ B（可选），`trigger_words.long_phase_transition` 有 8 个阶段切换词

## 九 + 短篇路径（短篇 5 模块）

> 仅当 length_type === "short" 时验证。

### 短篇通用
- [ ] **触发词识别**：`"写短篇"` / `"千字文"` / `"知乎体"` / `"番茄短篇"` 等能被正确识别为短篇路径
- [ ] **开场白**：先问长度（长/短）再展示模块菜单
- [ ] **5 模块菜单**：正确展示 ① 情绪锚点 ② 意象与极简规则 ③ 角色深潜（短篇版）④ 节拍设计 ⑤ 结尾类型学
- [ ] **不混入长篇模块**：在短篇路径中不应出现 ④ 故事脊梁 / ⑤ 黄金三章 / ⑥ 章节推演 / ⑦ 伏笔雷达 / ⑧ 收线结算 的 8 模块名称
- [ ] **5 层压缩**：受字数约束，问答密度合理（不一定走完 5 层）

### ① 情绪锚点
- [ ] 引导出 emotion.unity_of_effect（5-10 字复合短语）
- [ ] 引导出 hook_one_liner（≤30 字）
- [ ] 6 种钩子原型可被选择

### ② 意象与极简规则

- [ ] 不引导力量体系 10-15 级（短篇不需要）

### ③ 角色深潜（短篇版）

- [ ] 对手可以是抽象的（时间/命运/困境）

### ④ 节拍设计

- [ ] 触发中段（节拍 3-4）微转折检查

### ⑤ 结尾类型学

- [ ] 商业类短篇能引导 paywall_break_point
- [ ] 最后一击（动作/对话/意象/停顿）被设计

### 中途切换
