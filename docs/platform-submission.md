## 一、虾评 Skill 市场（P0 优先）

### 1. 注册与认证

- 访问：https://www.xiaopin.com/
- 注册开发者账号
- 完成实名认证（个人身份证 / 企业营业执照）

### 2. 创建 Skill

**步骤**：
1. 进入「Skill 市场」→「发布 Skill」
2. 填写基本信息：
   - Skill 名称：问道写作
   - 一句话描述：基于苏格拉底接生法的网文创作引导工具，帮作者想，不帮作者写
   - 详细描述：见下方文案
3. 上传 `SKILL.md` 作为主技能
4. 上传 `references/` 内容到知识库
5. 设置开场白（从 metadata.json 复制）
6. 设置推荐问题（4-6 个）
7. 上传封面图
8. 提交审核

### 3. 审核要点

虾评 Skill 市场审核通常关注：
- 内容合规性（无违规内容）
- 功能完整性（不是空白 Skill）
- 描述准确性（不能虚假宣传）
- 用户价值（确实能帮助用户）

**特别注意**：
- 「永远不生成正文」可能被误认为「功能残缺」 → 在描述中强调「引导式辅助」「创作教练」
- 引用 PRD §1.3 差异表（代写 vs 引导）作为差异化

### 4. 描述文案模板

```
问道写作 — 基于苏格拉底接生法的网文创作引导工具

【核心理念】
帮作者想，不帮作者写。
我们永远不替你写正文，只通过精准提问帮你想清楚。

【与代写工具的区别】
代写工具：生成正文 → 需人工修改 23-42%
问道写作：引导作者思考 → 输出大纲/人设/设定

【8 大创作模块】
① 选题锚定 → ② 世界观锻造 → ③ 角色深潜 → ④ 故事脊梁
→ ⑤ 黄金三章 → ⑥ 章节推演 → ⑦ 伏笔雷达 → ⑧ 收线结算

【适合谁】
- 网文新人（90% 的网文作者都是新人）
- 卡文作者（不知道怎么想下去）
- 想系统化创作的人
- 不想被 AI 代写污染文风的人

【输出】
对话中可导出 Markdown 格式的创作档案（选题/角色/大纲/伏笔）

【作者成长】
阶段一：详细引导（新人）
阶段二：减少提示（成长）
阶段三：独立创作（大师）
```

### 5. 关键词 SEO

为了让用户搜索到，标签必须覆盖：
- 网文、小说（最核心）
- 创作、写作（次核心）
- 苏格拉底（差异化）
- 大纲、人设（功能点）
- 世界观、伏笔（细分需求）

## 二、Coze 商店（P0 同步上架）

### 1. 注册与认证

- 访问：https://www.coze.cn/
- 注册账号并完成实名
- 进入「个人空间」→「发布 Bot」

### 2. 创建 Bot

**步骤**：
1. 创建 Bot：问道写作
2. 在「技能」配置项：
   - 添加 Skills → 上传 SKILL.md
   - 添加知识库 → 上传 references/*.md
3. 在「开场白」配置：复制 metadata.json 的 opening_message
4. 在「推荐问题」配置：从 suggested_questions 复制
5. 保存并发布

### 3. 审核要点

Coze 审核通常 1-3 天，关注：
- 内容安全
- 功能合规
- 描述真实

### 4. Coze 特有优化

Coze 平台支持插件调用，可以在 Bot 中：
- 启用「长期记忆」功能（变相手动 MCP 版持久化）
- 添加「用户画像」功能
- 配置多轮对话模板

## 五、应急预案

### 平台审核被拒

**可能原因**：
- 「不生成正文」= 功能残缺
- 内容合规问题
- 描述过于夸张

**应对**：
- 调整描述，强调「引导式辅助」「创作教练」
- 引用 PRD §1.3 差异表
- 提供详细使用案例
- 必要时申诉

### 用户大量反馈 Prompt 问题

**应对**：
- 紧急修复（直接改源文件 + 重新 assemble + 上传新版本）
- 在版本说明中标注修复内容
- 主动联系反馈用户告知已修复

### 平台政策变化

**应对**：
- 准备多平台备份（不依赖单一平台）
- 监控平台公告
- 建立社区（小红书/Discord）作为 Plan B

## 六、版本管理

### 版本号规则

```
v MAJOR . MINOR . PATCH

v1.0.0 - 初始版本
v1.0.1 - Bug 修复
v1.1.0 - 新功能（如新增模块）
v2.0.0 - 重大架构变更
```

### 更新流程

1. 修改 `skills/prompts/*.md` 或 `skills/references/*.md`
2. 运行 `node skills/assemble.mjs` 重新生成 SKILL.md
3. 更新 `skills/metadata.json` 的 version 字段
4. 在 `skills/SKILL.md` 顶部更新版本号
5. 平台更新 Skill 版本
