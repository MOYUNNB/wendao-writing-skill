## ✨ 问道写作是什么？

一个**只提问、不写正文**的网文 / 短篇创作引导工具。

把苏格拉底接生法引入创作——
像助产士帮助妇女生孩子一样，AI 通过**精准提问**帮作者"生产"自己的想法。

| | 代写工具（ChatGPT / Claude / 各种写手 AI） | 问道写作 |
|--|----------|---------|
| **做什么** | 帮你写正文 | 帮你想清楚 |
| **产出** | 一堆需要修改的文本 | 大纲、人设、设定、节拍、结尾 |
| **文风污染** | 严重（带 AI 味） | 无（你自己写） |
| **技能提升** | 越用越依赖 | 越用越会写 |
| **核心价值** | 短期产出 | 长期能力 |

**v1.1.0 新增短篇模式**：开篇先问"长篇/短篇"，自动加载 5 模块精简路径，覆盖知乎体、番茄短篇、小红书短篇、千字文等场景。

**v1.2.0 长篇两阶段拆分**：长篇 8 模块拆为 Phase A（写作前准备，必走）+ Phase B（写作手艺，可选）。完成 Phase A 后作者自选：进入 Phase B 打磨手艺 / 跳过直接拿 Story Bible 自主写正文。熟手友好，新手不亏。

**v1.3.0 资料检索**：Phase A 完成后 AI 主动建议联网搜时代资料（9 大方面：日常/服饰/建筑/语言/经济/科技/军事/民俗/禁忌），每条事实带来源，按"用到哪写到哪"原则写入对应 SB 字段。描写真实不悬浮。

## 🎯 双路径 + 长篇两阶段模块体系（v1.2.0）

v1.1.0 起，**开场白先识别作品类型**（长篇 / 短篇）。v1.2.0 起，长篇路径**进一步拆分为两阶段**——Phase A（写作前准备，必走）+ Phase B（写作手艺，可选）。熟手完成 Phase A 可跳过 Phase B 直接拿 Story Bible 自主写正文；新手可选 Phase B 打磨开篇/章节/伏笔/收尾。

### 为什么拆两阶段？

> 苏格拉底哲学 = 不居高临下、不替作者决策。

熟手本来就知道怎么写开篇/收尾，被强制过 Phase B 是浪费。新手才需要手艺指导。把选择权交回作者。

### 长篇路径 · 两阶段 8 模块（10 万字以上）

```
  ┌─────────────────────────────────────────────────────┐
  │  Phase A · 写作前准备（必走，做完就有 Story Bible）  │
  │  ① 选题锚定 → ② 世界观锻造 → ③ 角色深潜 → ④ 故事脊梁 │
  └────────────────────────┬────────────────────────────┘
                           ↓ ④ 完成后可选
                  ┌────────┴────────┐
                  ↓                 ↓
        ┌──────────────────┐  ┌──────────────┐
        │ 资料检索（v1.3.0+）│  │ 直接进 Phase B │
        │ 联网搜时代资料    │  │            │
        │ 带来源写入 SB     │  │            │
        └────────┬─────────┘  └──────┬───────┘
                 ↓                  ↓
        ┌─────────────────────────────┐    ┌────────────────────────┐
        │ Phase B · 写作手艺（可选）   │    │ 跳过 Phase B（熟手）    │
        │ ⑤ → ⑥ → ⑦ → ⑧               │    │ 直接导出 Story Bible   │
        │ 新手推荐走完                │    │ 自己写正文              │
        └─────────────────────────────┘    └────────────────────────┘
```

**Phase A · 写作前准备（必走）**：

| # | 模块 | 解决什么问题 | 触发词 |
|---|------|-------------|--------|
| ① [A] | **选题锚定** | 把模糊脑洞变成可验证的选题 | "我要开新书" |
| ② [A] | **世界观锻造** | 构建自洽、服务爽点的世界 | "设计世界" |
| ③ [A] | **角色深潜** | 创造有欲望、有缺陷的角色 | "添加角色" |
| ④ [A] | **故事脊梁** | 找到故事自己的节奏骨架 | "梳理大纲" |

**Phase B · 写作手艺（可选）**：

| # | 模块 | 解决什么问题 | 触发词 |
|---|------|-------------|--------|
| ⑤ [B] | **黄金三章** | 用 3 章锁住读者不弃书 | "设计开篇" |
| ⑥ [B] | **章节推演** | 为每章定义目标、冲突、钩子 | "规划章节" |
| ⑦ [B] | **伏笔雷达** | 管理伏笔的埋设与回收 | "埋伏笔" |
| ⑧ [B] | **收线结算** | 确保所有线索回收、角色弧光完成 | "写完了" |

**阶段切换触发词**（仅长篇流程中识别）：
- 进入 Phase B：`"进入 Phase B"` / `"继续 Phase B"` / `"下一步写正文"` / `"我想写正文"`
- 跳过 Phase B：`"跳过 Phase B"` / `"只导出大纲"` / `"我自己写"` / `"Phase A 导出"`
- 资料检索（v1.3.0+）：`"搜资料"` / `"查一下"` / `"搜点资料"` / `"查资料"` / `"[朝代/时代]资料"`（如"宋朝资料"）/ `"时代背景"` / `"真实情况"`

### 短篇路径 · 5 模块（千字-3 万字）

```
  ① 情绪锚点 → ② 意象与极简规则 → ③ 角色深潜（短篇版）
                                              ↓
                                       ④ 节拍设计 → ⑤ 结尾类型学
```

| # | 模块 | 解决什么问题 | 触发词 |
|---|------|-------------|--------|
| ① | **情绪锚点** | 锁定主导情绪 + 一句话钩子 | "写短篇" |
| ② | **意象与极简规则** | 2-3 意象 + 1-2 规则即冲突 | "短篇设定" |
| ③ | **角色深潜（短篇版）** | 1-3 人 + 一句话人设 + "能否砍掉" | "短篇角色" |
| ④ | **节拍设计** | 500-1000 字节拍 + 情绪弧线 | "短篇结构" |
| ⑤ | **结尾类型学** | 5 种结尾策略（反转/留白/圆形/开放/截断）| "短篇结尾" |

**自动识别触发词**：

- 长篇：`"开新书"` / `"新长篇"` / `"连载"` / `"网络小说"` ...
- 短篇：`"写短篇"` / `"千字文"` / `"知乎体"` / `"番茄短篇"` / `"微小说"` / `"这篇 5000 字"` ...

**模块可独立进入**——不需要走完全流程，从任何模块开始都行。

## 🌐 资料检索（v1.3.0+，仅长篇）

> 通用 AI 凭印象编"宋朝人都用 X"是网文最常见的悬浮病。问道写作 v1.3.0 起在 **Phase A 完成后**主动建议 AI 联网搜时代资料——用真实数据喂你的描写。

### 触发方式

- **自动**：④ [A] 故事脊梁 GRADUATE 后，AI 主动问"要不要先搜点资料？"
- **手动**：用户随时可说 `搜资料` / `查一下` / `搜点资料` / `宋朝资料` / `时代背景` / `真实情况`

### 9 大检索方面

| 方面 | 搜什么 | 古代示例（宋朝） |
|------|--------|-----------------|
| 日常 | 起居/集市/物价 | 瓦子夜市、交子铜钱、更点计时 |
| 服饰 | 官服/民服/织物 | 圆领袍、褙子、翘头履 |
| 建筑 | 民居/官衙/园林 | 抬梁木构、合院、瓦色分等级 |
| 语言 | 称谓/敬语/俗语 | 官职称谓、敬语系统 |
| 经济 | 货币/物价/税收 | 铜钱+交子、商税 |
| 科技 | 器物/工艺/历法 | 活字印刷、指南针、历法 |
| 军事 | 武器/兵制/阵法 | 弓弩、厢军、神臂弓 |
| 民俗 | 节庆/礼仪/信仰 | 元宵、祭祖、佛教道教 |
| 禁忌 | 避讳/礼制 | 帝名避讳、尊卑礼制 |

### 资料写入规范

每条事实**必须带来源**：

```
- 事实：宋朝女官常服为褙子配裙
  来源：[中国古代服饰史](https://...) · 抓取于 2026-06-26
```

AI 不会**凭印象补**——搜不到就明说"未找到可靠来源"。**搜来事实按用到哪写到哪**原则写入 SB 对应字段（world.costumes / world.architecture / characters.background / 等），不单独建 research_notes 字段。

### 严守原则

- **绝不写正文**（搜来事实不变成段落描写）
- **绝不编造来源**（搜不到就明说"未找到"）
- **总轮次 ≤ 3**（不无限深挖）
- **一次最多 2 方面**（控 token）

### 苏格拉底哲学保持

搜什么 → 你选；搜的结果 → 你确认入 SB；不替你做决定。

### 资料档案独立导出

积累足够多研究资料后，可单独导出独立 MD 文件，方便写作时查阅：

- **触发词**：`"导出研究"` / `"导出资料"` / `"导出档案"` / `"研究导出"` / `"资料导出"` / `"档案导出"`
- **或自动建议**：主 SB 导出时若含 ≥ 3 条带 source 的事实，AI 提示"是否同步导出研究档案.md？"
- **格式**：按 9 大方面归类（服饰/建筑/日常/语言/经济/科技/军事/民俗/禁忌 + 人物备注 + 剧情备注）
- **每条带**：`{name, url, fetched_at}` + **可信度分级**（高/中/低）
- **只列带 source 的事实**：不混入纯用户填写的设定
- **独立 MD**：与主 SB.md 分开保存

详细见 `skills/references/09-research.md` 和 `references/export-template.md` 的"资料档案导出模板"段。

## 🧠 工作原理：苏格拉底循环

每次对话都走这 5 阶段：

```
┌─────────────────────────────────────────┐
│  LISTEN  →  接收作者回答                 │
│     ↓                                    │
│  ASSESS  →  评估具体性/一致性/深度       │
│     ↓                                    │
│  DECIDE  →  选择 6 种分支动作之一        │
│     ↓                                    │
│  RESPOND →  生成回应（最多 3 个问题）    │
│     ↓                                    │
│  UPDATE  →  内部更新会话 Story Bible     │
└─────────────────────────────────────────┘
```

### 6 种分支动作

| 动作 | 何时触发 | 做什么 |
|------|---------|--------|
| **DEEPEN** | 回答具体但不够深 | 下一层追问 |
| **CONFIRM** | 回答够具体 | 复述确认后存储 |
| **CHALLENGE** | 回答与已有矛盾 | 指出矛盾，让作者选择 |
| **UNBLOCK** | 作者卡住/含糊 | 给 2-3 个选择题 |
| **PIVOT** | 当前维度挖透 | 切换问题方向 |
| **GRADUATE** | 必填字段全完成 | 总结，进入下一模块 |

### 5 层深挖（每模块通用）

```
第1层 表面 → 这个[角色/设定]最想让什么发生？
第2层 动机 → 为什么？这个欲望的根源是什么经历？
第3层 矛盾 → 想要的 vs 真正需要的——一样吗？
第4层 极限 → 保护这个欲望需要背叛最重要的人，他怎么选？
第5层 代价 → 致命弱点是什么？反派会怎么利用？
```

不是每次都必须走完 5 层。AI 根据回答的具体性动态决定深度。**短篇路径可走 3 层压缩版**（表面/动机/代价）。

## 💡 设计哲学

### 4 条不可违反的核心原则

1. **永远不生成正文** — 不写段落、对话、场景描写
2. **永远只提问** — 提问、追问、反问、确认
3. **每次回复最多 3 个问题** — 避免信息过载
4. **不居高临下** — 像教练而非裁判

### 为什么只提问不写？

> "代写工具让你**写得更轻松**，问道写作让你**写得更会**。"
>
> —— 一个用过 ChatGPT 写了 30 万字然后全部废稿重写的网文作者

写网文是**手艺活**。AI 代写就像外卖——方便但永远不是你自己的手艺。
问道写作是**私教**——不替你跑，但让你跑得越来越快。

**为什么用 BSL 而不是 MIT？**
- MIT 太宽松——别人可以分叉出"问道写作 Pro"直接商用
- BSL 给我们 **4 年变现窗口**，同时 **4 年后开源积累社区**
- 类似 MariaDB / Sentry / HashiCorp 的成熟做法
